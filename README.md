# Recipe Builder

### Цель

Создать консольное приложение `Recipe Builder` для управления списком ингредиентов с поддержкой undo — каждая команда реализована как чистая функция-модуль.

---

## Структура проекта

```
project/
  src/
   main.js               # обработка команд и history (undo)
   commands/
    add.js
    remove.js
    replace.js
    exists.js
    list.js
    unique.js
    sort.js
    merge.js
    find.js
    stringify.js
    undo.js
  package.json
  README.md
```

---

### Задачи

1. `add`

- Сигнатура: `add(recipe, ingredient)`
- Описание: вернуть новый массив с `ingredient`, добавленным в конец; не мутировать входной `recipe`.

2. `remove`

- Сигнатура: `remove(recipe, ingredient)`
- Описание: вернуть новый массив без первого вхождения `ingredient`; если не найдено — вернуть исходный массив.

3. `replace`

- Сигнатура: `replace(recipe, oldVal, newVal)`
- Описание: вернуть новый массив, где первое вхождение `oldVal` заменено на `newVal`.

4. `exists`

- Сигнатура: `exists(recipe, ingredient)`
- Описание: вернуть `true|false`; как минимум в одном месте использовать явный цикл `for`.

5. `list`

- Сигнатура: `list(recipe)`
- Описание: вернуть строку с пронумерованными ингредиентами или массив строк для вывода.

6. `unique`

- Сигнатура: `unique(recipe)`
- Описание: вернуть новый массив без дубликатов; не использовать `Set`.

7. `sortRecipe`

- Сигнатура: `sortRecipe(recipe, direction)`
- Описание: вернуть новый массив, отсортированный по `direction` (`'asc'|'desc'`).

8. `merge`

- Сигнатура: `merge(recipe, csvList)`
- Описание: распарсить `csvList` и вернуть объединённый новый массив.

9. `find`

- Сигнатура: `find(recipe, substring)`
- Описание: вернуть массив совпадений по подстроке (без учёта регистра).

10. `stringify`

- Сигнатура: `stringify(recipe)`
- Описание: вернуть строку `'a, b, c'` через `join`.

11. `undo`

- Сигнатура: `undo(history)`
- Описание: вернуть предыдущее состояние из `history` (реализация взаимодействует с `main.js`).

---

### Технические требования

- **Чистые функции:** все модули не мутируют входной `recipe`.
- **Возврат значений:** функции возвращают новые массивы/значения.
- **Синтаксис:** `const`/`let`, стрелочные функции.
- **Ограничения:** не использовать внешние библиотеки и `Set`/`Map`.
- **History:** `main.js` управляет `history` (push перед изменением, pop для undo).
